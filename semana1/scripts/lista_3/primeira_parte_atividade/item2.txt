2. Pesquise sobre a diferença entre exceções verificadas e não verificadas em Java. Dê exemplos de cada uma.

Em Java, as exceções são divididas em duas categorias: exceções verificadas (checked exceptions) e exceções 
não verificadas (unchecked exceptions).

Exceções Verificadas (Checked Exceptions): São aquelas que o compilador exige que sejam tratadas pelo código 
ou declaradas no método através da cláusula throws. Exemplos incluem IOException e SQLException.

Exemplo:

import java.io.File;
import java.io.FileNotFoundException;
import java.util.Scanner;

public class ExemploCheckedException {
    public static void main(String[] args) throws FileNotFoundException {
        File file = new File("arquivo.txt");
        Scanner scanner = new Scanner(file);
    }
}


Exceções Não Verificadas (Unchecked Exceptions): São exceções que o compilador não exige tratamento ou declaração. 
Geralmente, são subclasses de RuntimeException. Exemplos incluem NullPointerException e ArrayIndexOutOfBoundsException.

Exemplo:

public class ExemploUncheckedException {
    public static void main(String[] args) {
        String str = null;
        System.out.println(str.length());  // Isso causará NullPointerException
    }
}
